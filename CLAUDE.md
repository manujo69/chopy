## Repository structure

This is a monorepo with two independent, separately-installed Node projects (no shared root `package.json` or workspace config — `cd` into each before running its commands):

- `chopy-back/` — a minimal Express (v5) server, ESM (`"type": "module"`), single file `index.js`.
- `chopy-front/` — an Angular 20 application generated by Angular CLI, using standalone components, SSR (`@angular/ssr`), and the new `@angular/build:application` builder.
- `docs-data.json` (repo root) — a static copy of the OpenAPI spec for [FakeStoreAPI](https://fakestoreapi.com/docs), kept as a reference for the shape of `/products`, `/carts`, `/users`, and `/auth/login` endpoints the app is expected to integrate with.

The project is at an early/scaffold stage: `chopy-front` still contains the default Angular CLI placeholder template (`app.html`) and empty routes (`app.routes.ts`), and `chopy-back` is a single-endpoint CORS-enabled proxy.

## chopy-back

Backend is a thin proxy: `GET /` fetches `https://dummyjson.com/products` and returns the JSON as-is, with permissive CORS headers added for the Angular dev server. It listens on port `3000`.

```bash
cd chopy-back
npm install
node index.js        # starts the server on http://localhost:3000
```

There is no test suite configured (`npm test` is a placeholder that exits with an error) and no linter.

## chopy-front

Standard Angular CLI commands, run from `chopy-front/`:

```bash
cd chopy-front
npm install
npm start             # ng serve, dev server at http://localhost:4200
npm run build         # ng build (production by default)
npm run watch         # ng build --watch --configuration development
npm test              # ng test (Karma + Jasmine, launches Chrome)
```

Run a single test file/suite by passing standard Karma/Jasmine filters, e.g. `ng test --include='**/app.spec.ts'`.

SSR:
- `npm run build` produces both browser and server bundles (server entry: `src/server.ts`, SSR config: `src/main.server.ts` / `app.config.server.ts`).
- `npm run serve:ssr:chopy-front` runs the built SSR server from `dist/chopy-front/server/server.mjs` (build first).

No e2e framework is configured. No linter/formatter is configured beyond the `prettier` key in `package.json` (printWidth 100, single quotes, Angular parser for `*.html`) — there's no lint script wired up, so run `npx prettier --check .` / `--write .` directly if formatting is needed.

### Architecture

- Bootstrapping: `src/main.ts` (browser) and `src/main.server.ts` (server) both bootstrap the root `App` component (`src/app/app.ts`) using `appConfig` (`src/app/app.config.ts`), which wires up the router, zone-based change detection, and client hydration with event replay. `app.config.server.ts` merges in server-specific providers.
- Routing: `src/app/app.routes.ts` currently has no routes defined — add routes here as pages are built out.
- The app currently has no feature modules/components beyond the CLI-generated root `App` component — there is no existing pattern to follow yet for HTTP calls, state, or component structure. When adding the first real feature, decide conventions (e.g. `HttpClient` usage, folder layout) rather than assuming one from existing code.
